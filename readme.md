### 一、问题定义

我们的产品是一个应用于户外旅行场景的自行车车载仪。它将基
于Arduino 环境进行开发，预计实现播放音乐、蓝牙连接、超声波测
距、防盗及更多为单车旅行提供便利的功能。

 ### 二、市场调研


根据我们所选题目，小组成员对其进行了市场调研。首先我们在
百度及京东等购物平台进行了搜索，发现汽车、乃至摩托车的车载仪
器品类都较多。可是，针对自行车的车载仪却并不常见，这一现象与
日常生活所见完全一致。于是我们便分析其原因，发现其主要是由于
市场上产品种类较为单一且由于自行车相对较小不利于安装固定等
问题，导致人们对于自行车的车载仪接受度不高。 
市场现有产品有：



1. 车载GPS 定位器：它通常能定位位置，具有一定的防盗功能 2. 自行车时速表：可以显示速度、温度里程等信息
  接下来我们对于现有产品进行了分析，发现它们有 GPS 定位功
  能、时速显示、行驶里程等，但是通常都不是同时涵盖。这为我们的
  制作打开了一些思路，我们可以选择其中有用的部分并且加以整合。
  经过调查，我们发现其安装的可操作性与其防盗性也是用户考察的重
  要因素。 

  

  总结：如今自行车车载仪器相对较少，且功能较为单一。我们所
  选题目致力于解决这一实际问题

### 四、团建计划

#### (一) 问题定义



由于市面上针对自行车的车载仪器很少，我们预期设计一种自行
车车载仪，使其具有播放音乐，显示定位，超声波测距，蓝牙连接及
防盗等多种功能，同时保证其体积小巧且便于安装。由此来达到为自
行车用户的旅途提供便利的效果。此产品将基于Arduino环境进行开
发。 



#### (二) 可替代想法的产生



在产品设计的过程中，我们汇总了以下问题：

1. 定位准确性问题：采用新型芯片，减少网络延迟及定位误差

2. 蓝牙连接距离及稳定程度：采用更好性能的蓝牙模块

3. 产品体积及形状：尽可能做到小巧并保证减小阻力

4. 车辆防盗警告：警铃（超声波或内置电线法报警）及手机推送

5. 安装的稳定性及用户安装的难度：采用螺钉旋紧的方式或加装防
  滑垫。

6. 充电形式：电池、充电器、太阳能、动力传输

7. 防水防尘性能：采用闭合的塑料外壳

   #### (四) 详细设计

   1、准备好材料和工具

   2、将开发板连接到 GPS 模块上

   将导线焊接到 GPS 模块的引脚上，使用双面胶带，将 GPS 连接
   到 Arduino 板的底部，沿着 Arduino 板的底部折叠黑色线，将电阻
   支脚穿过 Arduino 板的引脚并焊接到 GPS 模块的引脚上。

   3、测试 GPS 模块

   在Arduino IDE上打开串口监视器，您现在应该能够看到从
   GPS 模块传输到Arduino板的数据。 

   ![lol](https://upload-images.jianshu.io/upload_images/22452548-59e501fd1db650b3.png?imageMogr2/auto-orient/strip|imageView2/2/w/1240)

   4、焊接无线

   5、准备一个接收器

   在VCC 和GND 之间焊接好导线，准备进行无线通信。 

   6、测试无线电

   将接收器插入电脑 USB,启动 Arduino IDE 并打开串行监视器,将 串行监视器设置为9600 bps 并且“无行结束”,键入“tx”并单击“发送”,如 果收到来自GPS 数据的消息记录器说明测试正常！

   7、蜂鸣器报警功能

   蜂鸣器模块有三个管脚，用三根杜邦线把模块接到开发板上，其
   中“-”脚接GND, 中间管脚接5V, "S"脚 接 D3口 ( D3 口可以用
   作 PWM）。 

   8、代码实现

   在arduino 中编写好代码上传到开发板。 
   串口速率与GPS模块要设置一致，接线方式如下： 

   ```c
   //TFT SDA -> UNO 11 
   //TFT SCK -> UNO 13 
   #define TFT_CS  10  // Chip select line for TFT display 
   #define TFT_A0   9  // Data/command line for TFT 
   #define TFT_RST  8  // Reset line for TFT (or connect to 
   +5V) 
   #define TFT_LED  A0 
   #define SD_CS    4   
   
   // Chip select line for SD card 
   //SD MISO -> UNO 12 
   //SD SCK -> UNO 13 
   //SD MOSI -> UNO 11 
   //GPS tx - UNO rx 
   //GPS rx - UNO tx 
   #define BTNZOOMIN 2 
   #define BTNZOOMOUT 3 
   ```

   #### (五) 设计辩护

   意义一：帮助使用者导航与定位，不致迷失方向

   伴随人们生活质量的提高，GPS 服务将逐步应用到私人旅游中， 比如到风景秀丽的地区去旅游，安装于车上的 GPS 接收机将充分发
   挥其全球定位的功能， 成为驾驶者最忠实的向导。在骑车浏览风景 的途中，乘车者可以随时知道车辆所在位置及行走速度和方向，从而
   避免迷失路途。

   意义二：帮助使用者防盗

   我们小组的GPS 记录仪不仅有导航功能，也有防盗功能。比如， 
   当自行车因不正常行为（如未被解锁）而被移出车库时，GPS 定位仪
   将会检测到其位置异常，从而发出报警信号。

   #### (七) 性能评估

    性能评估的测试主要分为两阶段。一是个体绩效的测试，我们对
   我们制造的产品进行单独的测试，验证它有能力做什么；二是测试我
   们的产品在户外旅行场景中与其他产品的交互能力，测试其是否能很
   好地融入该应用场景，并配合其他产品实现更好的单车户外旅行体验。 

    单独测试我们的产品时，我们主要采用的方法有用户实际体验以
   及对比同类型产品。我们可以邀请一些人员搭载我们的产品进行小型
   的单车户外骑行，验证其设计功能是否有用武之地。同时，人员们还
   会体验同类型产品以产生比较，通过这种方式，我们可以发现我们产
   品的一些不足和值得改进的地方，同时寻找我们产品的相对优势作为
   汇报时的亮点。而配合其他户外旅行场景产品进行测试，则能更好地
   验证我们产品的实用性。
